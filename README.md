# Nand To Tetris

## Introduction
Nand To Tetris는 각 Chapter마다 두가지를 매우 강조한다.
1. Abstraction(=Interface).
	저자가 말하는 Abstraction이란, 해당 개념 혹은 기술이 무엇을(What)하는지에만 관점을 둔 행위를 말한다.
	만약, 우리가 어떤 기술의 이용자라면 구현(Implementation)은 신경쓰지 않고, 해당 기능이 잘 동작할 것 이라는 믿음을 가지고 추상화된 Interface만 이용하면 된다. 
2. Implementation
	저자가 말하는 Implementation이란, 해당 기술 혹은 기능이 제대로 동작하기 위한 세부사항을 만족시키며 내부 구성을 생성한 것을 의미한다.
	말이 어려운데, 결국 해당 기능이 잘 동작하기 위해서 필요한 것들을 모두 갖춘 결과물이라고 생각하면 될 것 같다.
	이용자 측면에서 이러한 구현을 알 필요가 없다.

위의 두가지 개념을 가지고 전체적인 맥락을 소개한다.
매 Chapter마다 각 개념의 정의를 소개하고, 해당 개념의 추상화(어떤 작업을 하는지)를 설명한다. 이후, 이러한 작업을 위한 명세(세부사항)을 구현하기 위한 설명을 진행하는 방식의 구성이다.

위에서 말한 것들을 숙지하고 이제 본 내용에 들어가보자.
